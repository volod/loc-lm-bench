Ти аналітик, що будує онтологію з україномовного документа для оцінювання RAG.
Виокреми з тексту нижче (нічого не вигадуй; усе має спиратися на текст):
1. named entities -- name, type, aliases, mentions. Тип ОБОВ'ЯЗКОВО один із набору (якщо не підходить жоден -- став MISC): {{ entity_types }};
2. events -- короткий опис + evidence;
3. claims -- твердження + evidence;
4. facts -- трійки subject-relation-object + evidence.
Кожне поле evidence та кожен елемент mentions МАЄ бути дослівною цитатою з тексту (скопіюй точно, символ у символ).
Поверни лише JSON-об'єкт з ТОЧНО такими ключами верхнього рівня: "entities", "events", "claims", "facts".
Не використовуй ключ "relations": відношення записуй тільки у "facts".
Якщо для якогось списку немає елементів, поверни порожній список [].
Обмеження розміру: максимум 8 entities, 4 events, 4 claims, 8 facts.
Пріоритет: спочатку facts і claims з явним evidence, потім найважливіші entities; не виписуй довгі переліки назв, якщо через це бракує facts.
Схема:
{"entities": [{"name": ..., "type": ..., "aliases": [...], "mentions": [<цитата>]}],
 "events": [{"description": ..., "evidence": <цитата>}],
 "claims": [{"text": ..., "evidence": <цитата>}],
 "facts": [{"subject": ..., "relation": ..., "object": ..., "evidence": <цитата>}]}
